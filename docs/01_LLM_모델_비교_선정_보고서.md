# LLM 모델 비교·선정 보고서

## 1. 비교 목적

본 보고서는 동일한 업무 과업인 **AI 뉴스 큐레이터(뉴스 자동화)**를 수행하기 위해 GPT 5.5, Claude Sonnet 4.6, Gemini 2.5 Flash를 대상으로 성능을 비교하고, 뉴스 자동화 업무에 가장 적합한 LLM을 선정하는 것을 목적으로 한다.

비교는 동일한 테스트 입력을 사용하여 수행하였으며, 뉴스 선별 정확성, 사실성, 형식 준수, 문맥 유지 등 실무 활용에 필요한 항목을 기준으로 평가하였다.

---

# 2. 업무 과업

## 업무명

**AI 뉴스 큐레이터 (News Automation AI)**

## 업무 내용

여러 뉴스 기사 중 국민 생활에 영향을 주는 핵심 뉴스를 선별하고, 중복 기사를 제거한 뒤 객관적인 요약을 생성한다.

출력은 다음 형식을 따른다.

* 제목
* 선정 이유
* 핵심 요약
* 국민에게 미치는 영향
* 확인 필요

# 4. 동일 테스트 입력

모든 모델에 동일한 테스트 입력을 적용하였다.

### Test 1

뉴스 기사 목록에서 국민 생활에 가장 중요한 뉴스 3개를 선정한다.

---

### Test 2

---

### Test 3

---

### Test 4

---

### Test 5

# 7. 모델별 분석

## GPT 5.5

GPT 5.5는 뉴스 자동화 업무에서 가장 안정적인 결과를 생성하였다.

특히 출력 형식 준수, 문맥 유지, 시스템 프롬프트 반영 능력이 우수하였으며, 기사에 없는 내용을 생성하지 않고 "확인 필요"를 적절하게 표시하였다. 또한 시스템 프롬프트, Few-shot 예시, 환각 테스트, 실행 로그까지 일관성 있게 설계할 수 있어 실무 적용성이 높았다.

---

## Claude Sonnet 4.6

Claude Sonnet 4.6은 프롬프트 개선 방향을 분석하는 능력이 뛰어났다.

확인 질문 조건을 실제 예외 상황 중심으로 축소해야 한다는 점, 뉴스 중요도 기준에 우선순위를 부여해야 한다는 점, Markdown 또는 JSON 구조화를 통해 자동화 파이프라인과의 연동성을 높일 수 있다는 점을 제안하였다.

환각 테스트에서는 대부분의 항목을 통과하였으나, 제목만으로 동일 사건 가능성을 추정하는 부분은 조건부 PASS로 평가하였다.

---

## Gemini 2.5 Flash

Gemini 2.5 Flash는 프롬프트 개선 과정을 체계적으로 분석하였다.

단계적 접근(Reasoning Prompt), 확인 질문, 출력 구조화, 자동화 친화적인 설계가 뉴스 자동화 업무에 효과적이라는 점을 제시하였다. 또한 입력 검증 강화와 Markdown 구조화를 통해 API 및 자동화 시스템과의 연계성을 높일 수 있다는 개선 방향을 제안하였다.

---

# 8. 프롬프트 개선 과정

본 과제에서는 하나의 모델만 사용하지 않고 **GPT 5.5를 중심으로 Claude Sonnet 4.6과 Gemini 2.5 Flash의 피드백을 반복적으로 반영**하는 방식으로 프롬프트를 개선하였다.

초기 v1 프롬프트는 뉴스를 단순 요약하는 수준이었으나, v2에서는 단계적 접근, 확인 질문, 사실 검증 규칙을 추가하였다. 이후 Claude Sonnet 4.6의 피드백을 반영하여 중요도 우선순위, 예외 처리, 출력 구조를 개선하였고, Gemini 2.5 Flash의 피드백을 반영하여 입력 검증 강화와 자동화 친화적인 Markdown 구조를 적용하였다.

이와 같은 반복 개선(Iteration)을 통해 단일 모델만 사용할 때보다 프롬프트의 안정성과 재현성을 높일 수 있었다.

---

# 9. 최종 선정 모델

최종 모델은 **GPT 5.5**로 선정하였다.

선정 이유는 다음과 같다.

1. 뉴스 자동화 업무에서 가장 높은 정확도와 안정성을 보였다.
2. 긴 대화에서도 조건 변경과 출력 형식 변경을 일관되게 유지하여 문맥 유지 능력이 가장 우수하였다.
3. 기사에 없는 내용을 생성하지 않고 "확인 필요"를 표시하여 환각을 효과적으로 방지하였다.
4. 시스템 프롬프트, Few-shot, 환각 테스트, 실행 로그를 포함한 전체 문서를 가장 일관성 있게 생성하였다.
5. Claude Sonnet 4.6과 Gemini 2.5 Flash의 피드백을 반영하여 프롬프트를 반복 개선함으로써 뉴스 자동화 업무에 적합한 최종 프롬프트를 완성할 수 있었다.

---

# 10. 결론

본 실험에서는 GPT 5.5, Claude Sonnet 4.6, Gemini 2.5 Flash를 대상으로 동일한 뉴스 자동화 업무를 수행하여 성능을 비교하였다.

비교 결과, GPT 5.5는 뉴스 선별 정확성, 문맥 유지, 형식 준수, 자동화 적합성에서 가장 우수한 성능을 보였으며 최종 모델로 선정하였다. Claude Sonnet 4.6은 프롬프트 구조 개선과 예외 처리 규칙에 대한 구체적인 피드백을 제공하였고, Gemini 2.5 Flash는 단계적 접근과 자동화 친화적인 출력 구조를 제안하여 프롬프트 개선에 기여하였다.

본 과제에서는 단일 모델의 결과만 사용하는 것이 아니라 여러 LLM의 피드백을 반복적으로 반영하는 방식으로 프롬프트를 개선하였다. 이를 통해 뉴스 자동화 업무에서 높은 안정성과 재현성을 확보할 수 있었으며, 실제 업무 환경에서도 활용 가능한 AI 뉴스 큐레이터 시스템을 설계할 수 있었다.
